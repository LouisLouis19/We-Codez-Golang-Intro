Overview

This week, we will explore advanced applications of Python data structures. You will learn about nested data structures, how to use comprehensions to create and manipulate data, and how to choose the right data structure based on the use case.

Nested Data Structures

Nested data structures allow you to create complex data models by combining different data types. You can have lists within lists, dictionaries of lists, or any other combination.

Lists of Lists

A list of lists is simply a list where each element is itself a list. This can be useful for representing matrices, tables, or any other data that has a grid-like structure.

# Example: List of lists representing a 3x3 matrix
matrix = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
]

# Accessing an element
print(matrix[0][1])  # Outputs 2

Dictionaries of Lists

A dictionary of lists is a dictionary where each key maps to a list. This structure is useful for storing collections of items that are categorized by a common key.

# Example: Dictionary of lists representing students and their grades
grades = {
    "John": [85, 90, 78],
    "Alice": [88, 92, 85],
    "Bob": [70, 75, 80]
}

# Accessing and manipulating a list within the dictionary
grades["John"].append(95)
print(grades["John"])  # Outputs [85, 90, 78, 95]

Accessing and Manipulating Nested Structures

To access or manipulate elements in nested data structures, you use multiple indices or keys. The first index/key accesses the outer structure, and the subsequent indices/keys access the nested structures.

# Example: Accessing and modifying a value in a nested dictionary
school = {
    "class1": {"teacher": "Mrs. Smith", "students": ["John", "Alice"]},
    "class2": {"teacher": "Mr. Brown", "students": ["Bob", "Charlie"]}
}

# Accessing a nested value
teacher_class1 = school["class1"]["teacher"]

# Modifying a nested value
school["class2"]["students"].append("Dave")
print(school["class2"]["students"])  # Outputs ['Bob', 'Charlie', 'Dave']

Comprehensions

Comprehensions provide a concise way to create and manipulate collections in Python. They can be used to generate lists, dictionaries, and sets in a single line of code.

List Comprehensions

List comprehensions allow you to create a new list by applying an expression to each item in an existing collection.

# Example: Creating a list of squares using list comprehension
squares = [x**2 for x in range(10)]
print(squares)  # Outputs [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

Dictionary Comprehensions

Dictionary comprehensions work similarly to list comprehensions, but they generate key-value pairs instead of just values.

# Example: Creating a dictionary of squares using dictionary comprehension
squares_dict = {x: x**2 for x in range(10)}
print(squares_dict)  # Outputs {0: 0, 1: 1, 2: 4, 3: 9, 4: 16, 5: 25, 6: 36, 7: 49, 8: 64, 9: 81}

Set Comprehensions

Set comprehensions create a set by applying an expression to each item in an existing collection.

# Example: Creating a set of unique squares using set comprehension
unique_squares = {x**2 for x in range(10) if x % 2 == 0}
print(unique_squares)  # Outputs {0, 16, 64, 4, 36}

Choosing the Right Data Structure

Choosing the right data structure depends on the specific needs of your program. Here are some guidelines:

  • Lists: Use when you need an ordered collection that allows duplicates and requires fast access by index.
  • Tuples: Use when you need an ordered, immutable collection. Tuples are also more memory-efficient than lists.
  • Dictionaries: Use when you need to map keys to values and require fast lookups by key.
  • Sets: Use when you need a collection of unique elements and need to perform fast membership tests.

Performance Considerations

Each data structure has its own performance characteristics. For example, lists and tuples allow fast access by index, while dictionaries and sets allow fast lookups by key. When choosing a data structure, consider the following:

  • Time Complexity: How long it takes to perform basic operations like access, insert, and delete.
  • Space Complexity: How much memory the data structure uses.

In general, dictionaries and sets provide O(1) average time complexity for lookups, while lists and tuples provide O(n) time complexity for searches but O(1) time complexity for access by index.

1. Work with Nested Data Structures

Create a list of dictionaries representing students, with each dictionary containing the student's name, age, and a list of grades. Access and modify elements within the nested structure.

Solution 
students = [
    {"name": "John", "age": 22, "grades": [85, 90, 78]},
    {"name": "Alice", "age": 21, "grades": [88, 92, 85]},
    {"name": "Bob", "age": 23, "grades": [70, 75, 80]}
]

# Accessing a student's name
print(students[0]["name"])  # Outputs 'John'

# Adding a grade to a student's list of grades
students[1]["grades"].append(95)
print(students[1]["grades"])  # Outputs [88, 92, 85, 95]

2. Write List Comprehensions

Write a list comprehension to create a list of squares of all even numbers between 1 and 20.

Solution 
even_squares = [x**2 for x in range(1, 21) if x % 2 == 0]
print(even_squares)  # Outputs [4, 16, 36, 64, 100, 144, 196, 256, 324, 400]

3. Write Dictionary Comprehensions

Write a dictionary comprehension to create a dictionary mapping numbers to their cubes for numbers 1 through 10.

Solution 
cubes_dict = {x: x**3 for x in range(1, 11)}
print(cubes_dict)  # Outputs {1: 1, 2: 8, 3: 27, 4: 64, 5: 125, 6: 216, 7: 343, 8: 512, 9: 729, 10: 1000}

4. Practice Set Comprehensions

Write a set comprehension to create a set of all unique vowels in a given string.

Solution 
string = "hello world"
vowels = {char for char in string if char in "aeiou"}
print(vowels)  # Outputs {'o', 'e'}

5. Choosing the Right Data Structure

Discuss a scenario where you need to store and frequently look up student records by their unique ID. Which data structure would be most appropriate and why?

Discussion

A dictionary would be most appropriate because it allows fast lookups by key (the student's ID). This makes it efficient for retrieving records quickly based on a unique identifier.
