Debugging Techniques

Print Statements (10 minutes)

Using print statements is a basic yet effective method for debugging. Insert print() statements in your code to output variable values and track the flow of execution.

Example

def divide_numbers(a, b):
    print("a =", a)
    print("b =", b)
    result = a / b
    print("result =", result)
    return result

# Example usage
divide_numbers(10, 2)

This will output the values of `a`, `b`, and `result` to help trace any issues.

Exercise

Given a small code snippet with bugs (e.g., a function that should calculate the average of a list but is giving incorrect results), use print() statements to debug and identify the issue.

Solution 
def calculate_average(numbers):
    total = 0
    for num in numbers:
        total += num
    average = total / len(numbers)
    print("total =", total)
    print("average =", average)
    return average

# Example usage
calculate_average([10, 20, 30])

Using Debuggers (15 minutes)

Integrated debuggers in IDEs like PyCharm or VSCode, and tools like pdb (Python Debugger) provide advanced debugging capabilities. You can set breakpoints, step through code, and inspect variable values.

Basic Commands

  • Set Breakpoints: Pause execution at specific lines to inspect code.
  • Step Over: Execute the next line of code, skipping over function calls.
  • Step Into: Enter the function call and debug inside it.
  • Continue: Resume code execution until the next breakpoint.
  • Inspect Variables: View the current values of variables.

Example

In VSCode, set breakpoints by clicking in the gutter next to the line numbers. Use the Debug sidebar to step through code and inspect variables.

Exercise

Use the VSCode debugger to step through a code snippet (e.g., a function that calculates factorial) and inspect the state of variables at each step.

Solution 
def factorial(n):
    result = 1
    for i in range(1, n + 1):
        result *= i
    return result

# Example usage
factorial(5)

Exception Handling (15 minutes)

Exception handling is crucial for managing errors gracefully. Use try, except, and finally blocks to catch and handle exceptions without crashing your program.

Example

def safe_divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("Error: Cannot divide by zero.")
        return None
    finally:
        print("Execution completed.")
    return result

# Example usage
safe_divide(10, 0)

This code handles division by zero and ensures that a message is printed regardless of whether an error occurs.

Exercise

Modify a code snippet that reads from a file to include exception handling for cases like file not found or permission issues.

Solution 
def read_file(filename):
    try:
        with open(filename, 'r') as file:
            content = file.read()
            return content
    except FileNotFoundError:
        print("Error: File not found.")
    except PermissionError:
        print("Error: Permission denied.")
    return None

# Example usage
read_file('nonexistent_file.txt')
