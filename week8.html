Overview

This week, we will explore Python dictionaries and sets, two powerful data structures that allow you to manage data in unique ways. You will learn how to create and manipulate these structures and understand their use cases.

Python Dictionaries

A dictionary is an unordered, mutable collection of key-value pairs. Each key is unique, and it is associated with a value. Dictionaries are perfect for storing data that you want to quickly look up by key.

Creating Dictionaries and Accessing Values by Key

Dictionaries are created using curly braces {}, with key-value pairs separated by colons. 

# Creating a dictionary
student = {"name": "John", "age": 22, "major": "Computer Science"}
print(student["name"])  # Accessing the value associated with the key 'name'

Basic Operations

  • Adding: Add a new key-value pair to the dictionary.
  • Updating: Update the value of an existing key.
  • Removing: Remove a key-value pair from the dictionary using del or the pop() method. 
# Adding and updating elements
student["graduation_year"] = 2024  # Adding a new key-value pair
student["age"] = 23  # Updating the value of an existing key

# Removing an element
del student["major"]  # Removes the key 'major' and its value
graduation_year = student.pop("graduation_year")  # Removes and returns the value associated with 'graduation_year'

Dictionary Methods

Python provides several built-in methods for dictionaries:

  • get(): Returns the value for a key, or a default value if the key does not exist.
  • keys(): Returns a view object of all keys in the dictionary.
  • values(): Returns a view object of all values in the dictionary.
  • items(): Returns a view object of all key-value pairs in the dictionary as tuples. 
# Using dictionary methods
name = student.get("name", "Unknown")  # Returns 'John'
keys = student.keys()  # Returns a view object of all keys
values = student.values()  # Returns a view object of all values
items = student.items()  # Returns a view object of all key-value pairs

Python Sets

A set is an unordered collection of unique elements. Sets are useful for storing items where duplicates are not allowed, and you need to perform fast membership tests.

Creating Sets and Basic Operations

Sets are created using curly braces {} or the set() function. 

# Creating a set
fruits = {"apple", "banana", "cherry"}
empty_set = set()  # Creating an empty set

Basic Operations

  • Adding: Add an element to the set using the add() method.
  • Removing: Remove an element using the remove() or discard() method. 
# Adding and removing elements
fruits.add("orange")  # Adds 'orange' to the set
fruits.remove("banana")  # Removes 'banana' from the set (raises KeyError if not found)
fruits.discard("pear")  # Removes 'pear' if it exists (no error if not found)

Set Methods

Python provides several built-in methods for sets:

  • add(): Adds an element to the set.
  • remove(): Removes an element from the set (raises an error if the element does not exist).
  • discard(): Removes an element from the set (does nothing if the element does not exist).
  • union(): Returns a new set with elements from both sets.
  • intersection(): Returns a new set with elements common to both sets.
  • difference(): Returns a new set with elements in the first set but not in the second. 
# Set operations
set_a = {"apple", "banana", "cherry"}
set_b = {"banana", "orange", "grape"}

union_set = set_a.union(set_b)  # {'apple', 'banana', 'cherry', 'orange', 'grape'}
intersection_set = set_a.intersection(set_b)  # {'banana'}
difference_set = set_a.difference(set_b)  # {'apple', 'cherry'}

Use Cases and Differences Between Sets and Lists

  • Uniqueness: Sets automatically remove duplicate elements, making them ideal for tasks like removing duplicates from a list.
  • Efficiency: Sets are optimized for membership tests, making them faster than lists for checking if an item exists.
  • Order: Lists maintain the order of elements, whereas sets do not.

Practice

1. Create and Manipulate Dictionaries

Create a dictionary representing a book with keys for title, author, and year. Add a key for genre, update the year, and remove the author.

Solution 
book = {"title": "1984", "author": "George Orwell", "year": 1949}
book["genre"] = "Dystopian"
book["year"] = 1950
del book["author"]
print(book)  # {'title': '1984', 'year': 1950, 'genre': 'Dystopian'}

2. Practice Using Dictionary Methods

Given a dictionary of student grades, print all the keys, values, and items in the dictionary.

Solution 
grades = {"John": "A", "Alice": "B", "Bob": "C"}
print(grades.keys())  # dict_keys(['John', 'Alice', 'Bob'])
print(grades.values())  # dict_values(['A', 'B', 'C'])
print(grades.items())  # dict_items([('John', 'A'), ('Alice', 'B'), ('Bob', 'C')])

3. Create and Manipulate Sets

Create a set of numbers, add a new number, and then remove a number. Print the final set.

Solution 
numbers = {1, 2, 3}
numbers.add(4)
numbers.remove(2)
print(numbers)  # {1, 3, 4}

4. Practice Using Set Operations

Given two sets, find their union, intersection, and difference.

Solution 
set1 = {"a", "b", "c"}
set2 = {"b", "c", "d"}

union_set = set1.union(set2)
intersection_set = set1.intersection(set2)
difference_set = set1.difference(set2)

print(union_set)  # {'a', 'b', 'c', 'd'}
print(intersection_set)  # {'b', 'c'}
print(difference_set)  # {'a'}

5. Apply Sets in Real-World Scenarios

Imagine you have two lists of email addresses, and you want to find the addresses that appear in both lists. Use a set to solve this problem.

Solution 
emails_list1 = ["a@example.com", "b@example.com", "c@example.com"]
emails_list2 = ["b@example.com", "c@example.com", "d@example.com"]

set1 = set(emails_list1)
set2 = set(emails_list2)

common_emails = set1.intersection(set2)
print(common_emails)  # {'b@example.com', 'c@example.com'}
